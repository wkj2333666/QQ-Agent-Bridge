# 图片生成、识图与视频理解

用于图片生成与绘制、图表、标注图片、识图、截图分析、B站/b23.tv/网页视频理解。

## Images

- 分析图片前必须读取用户附带的本地图片路径或可访问 URL。
- 生成图片、绘图、图表、标注图应保存到 outbox。
- 适合直接看的 PNG/JPG 用 `QQBOT_SEND_IMAGE`；源文件、SVG、PDF 图册用 `QQBOT_SEND_FILE`。

### Animated Images

- GIF、APNG、动画 WebP 是有时间顺序的多帧媒体，不是普通静态图。桥接层通常会同时提供原始动图、源帧数/时长，以及由 `ffprobe`、`ffmpeg` 有界提取的采样帧路径和时间点。
- 必须实际读取所有可用采样帧，按时间顺序比较人物、文字、动作和场景变化，再描述动图；首帧不能代表完整动图，也不能仅凭文件名、表情包文字或静态预览推测后续内容。
- 采样帧是有限证据：可以说明采样中看见的变化，但不能声称逐帧覆盖了未采样区间。循环播放不代表内容重复次数具有语义。
- 上下文标记“动态证据不可用”时，只能描述已实际读取的静态画面，并明确无法验证动态过程；不得把首帧当作整段 GIF/APNG/动画 WebP。
- Agent 不要再次无界抽帧。确需补充时，仍须限制时长、分辨率和帧数，并在结束后清理临时帧。

## Video

只按已取得的正片证据回答；本参考不增加下载器或视频后端。

### 证据边界

- 页面元数据（标题、作者、发布时间、时长、简介）用于识别资源，不等于直接内容证据。
- 直接内容证据是该视频的字幕、转写、音频、画面/抽帧或用户提供的实际片段。不能只凭标题、短链文本、搜索片段或常识推断正片内容。
- 相似主题文章、同名标题、新闻和背景资料不能当作视频内容证据，只能标为背景或“未验证为视频内容”。

### 取证流程

1. **视频来源**：确认用户给出的 URL、平台、视频页或本地附件；Bilibili/哔哩哔哩的 BV/AV视频页与用户提供的本地视频、本地附件可直接作为入口读取。
2. **解析短链**：对 `b23.tv` 或其他短 URL 展开重定向，确认最终视频页；抖音、快手、腾讯视频等中文视频平台也按“解析短链或打开页面，再读取元数据”的流程处理，不假定各平台能力相同。
3. **页面元数据**：读取最终页面可验证的标题、作者、时长、发布时间和简介，并记录页面访问是否成功。
4. **字幕优先**：先获取该页原生字幕；原生字幕没有时再检查人工字幕或自动字幕。取得字幕、转写、音频或画面等直接内容证据后，才总结或回答视频内容。
5. **媒体补证**：无字幕时，若有可访问音频，提取音频并用本地 Whisper 做语音识别/转写；需要画面信息时，必要时抽帧并查看多个时间段。只引用实际读到的字幕、转写或帧中信息。

### 长视频与本地媒体

- 长视频先用字幕定位；设置有限的 token 预算、时间戳和帧数预算，不做无界转写或抽帧。
- 用户点名某一刻、人物或情节时，优先围绕该时间窗口采样，必要时再扩展到多个时间段；每个结论保留对应时间戳。
- 本地视频、用户附带的视频或已下载附件优先直接读取；需要转写时提取单声道 16 kHz 音频供本地 Whisper 使用。
- 处理结束后清理临时下载、临时音频和抽帧文件；不要把临时路径或用户提供的 cookie 暴露在回答中。

### 访问受阻

- 出现登录、cookie 缺失、403、429、地域限制、限流或反爬阻塞时，不要绕过访问控制、不得伪造 cookie 或会话、不要总结视频内容；只列已验证元数据和阻塞点。
- 用户提供的 cookie 或已下载附件只能按桥接层或用户原样提供的范围使用，不展示、不猜测也不补造。
- 直接内容证据不足、无法取得正片时，使用阻塞回复：说明阻塞点、列出已验证元数据、说明需要的授权或附件；不要猜测或推断视频内容。

### 进度

- 仅在真实完成的阶段输出简短进度，例如 `QQBOT_PROGRESS: 已解析链接`、`QQBOT_PROGRESS: 字幕已读取` 或 `QQBOT_PROGRESS: 已抽帧`。
- 不要刷屏；失败尝试、计划中的步骤和内部细节不报作完成。

## Complete

视觉交付完成 = 内容证据或文件已验证；发图用 `QQBOT_SEND_IMAGE`，其他文件用 `QQBOT_SEND_FILE`。
