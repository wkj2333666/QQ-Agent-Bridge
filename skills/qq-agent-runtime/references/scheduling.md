# 自然语言定时任务语义

用于把 `/schedule` 后的自然语言编译为结构化时间规则。只理解，不执行任务。

## 语义分层

分别判断四件事，不要把它们混成一段文本：

1. **时间规则**：何时首次触发，是否重复，何时结束。
2. **发送目标**：真实 QQ mention 由 bridge 单独保存，不属于正文。
3. **动作类型**：`send`、`ask` 或 `task`。
4. **动作内容**：触发时真正发送或执行的内容。

## 动作选择

- `send`：每次发送完全相同的静态正文，不需要模型临场生成。
- `ask`：触发时让模型生成一条轻量回答，例如笑话或问候。
- `task`：触发时需要天气、搜索、网页、文件、汇总等工具或外部信息。

## send_text

`action=send` 时必须单独给出 `send_text`。它回答：

> bridge 已经构造好真实 @ 段后，紧接着应该显示哪一段正文？

- 时间表达、目标 @、命令措辞和叙述连接词不属于 `send_text`。
- 引号、书名号或明确的“内容是”所界定的文字属于正文；其中即使出现“并说”等字样也必须保留。
- `send_text` 必须是用户原文中的连续片段，不得改写或补字。
- 无法可靠区分正文时标记歧义，不要猜。

这是基于上下文、引号和用户意图的**语义分段**，不是按关键词删除前缀。

## Examples

| 用户原文 | action | payload / send_text |
| --- | --- | --- |
| `每过1分钟就 @1583165466 并说谢森同我爱你` | `send` | `谢森同我爱你` |
| `明天十点提醒我喝水 噔噔噔` | `send` | `喝水 噔噔噔` |
| `每天八点讲个冷笑话` | `ask` | `讲个冷笑话`；`send_text=null` |
| `每天八点告诉我北京市天气` | `task` | `告诉我北京市天气`；`send_text=null` |
| `每天八点说“并说这两个字很好玩”` | `send` | `并说这两个字很好玩` |

关键正例：

```text
输入：每过1分钟就 @1583165466 并说谢森同我爱你
输出：action="send", payload="谢森同我爱你", send_text="谢森同我爱你"
```

关键反例：

```text
输入：每天八点说“并说这两个字很好玩”
输出：action="send", payload="并说这两个字很好玩", send_text="并说这两个字很好玩"
```

## Self-check

输出前在心里模拟一次实际 QQ 消息：`<真实 @ 段> + send_text`。结果应当像用户想看到的消息，而不是创建定时任务时说的整句话。
