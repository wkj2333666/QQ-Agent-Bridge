# 语音、音频与唱歌

用于语音识别、语音生成、音频处理、唱歌、旋律、音色、指定声音风格。

## Speech Recognition

- 用户 QQ 语音会作为 `voice` 资源出现；必须读取或转写音频后再总结。
- 转写失败时说明失败原因，不要凭上下文补全语音内容。

## Voice Generation

- 只有生成的人声、短回复适合 QQ 语音条。
- `QQBOT_SEND_VOICE` 必须是可验证真实时长 <=60 秒，并写 `duration=<seconds>`。
- 泛音频、音乐、播客、较长音频、不确定时长或无法验证时长的音频，用 `QQBOT_SEND_AUDIO` 或 `QQBOT_SEND_FILE`。

## Singing

- 唱歌需要旋律线、音高变化、节奏和歌声音色；普通 TTS、朗读歌词、念白、只改变语速或加背景音乐的朗读不算唱歌。
- 必须显式发现并调用外部 singing backend、歌声生成后端、音乐生成服务或同等工具后，才可以声称能唱；`ffmpeg`、音频转码、TTS 或 QQ 发送接口本身不是唱歌能力。
- 若没有可用音乐/歌声生成工具，必须说明阻塞，不能假装已唱，也不能退化成 TTS 或用朗读 TTS 冒充。
- 能生成时，优先产出带旋律的人声/音乐音频文件，并用 `QQBOT_SEND_AUDIO` 按文件发送。
- 只有非常短、可验证真实时长 <=60 秒、且确实是带旋律的人声时，才可以考虑 `QQBOT_SEND_VOICE`；不确定时一律用 `QQBOT_SEND_AUDIO`。
- 成功发送歌曲/歌声音频时，最终答案只输出 `QQBOT_SEND_AUDIO` 或必要发送指令，不要附加说明文字、歌词全文、制作过程或“我已经唱好了”等多余文本。
- 用户指定音色时只做风格描述或使用可用工具参数；不要声称克隆真实人物声音，除非用户提供合法授权和工具支持。

## Complete

语音/唱歌完成 = 文件存在非空 + 时长/格式已检查 + 使用正确 QQBOT_SEND_* 指令。唱歌还必须是外部 singing backend 或歌声生成后端实际生成的带旋律歌声/音乐音频；朗读版 TTS、念白、背景音乐加朗读、音频转码都不是完成。
